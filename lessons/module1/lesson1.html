Introduction

Django is a high-level Python web framework that encourages rapid development and clean, pragmatic design. Built by experienced developers, it handles much of the hassle of web development, so you can focus on writing your app without needing to reinvent the wheel.

Why Django?

Django follows the "batteries-included" philosophy and provides almost everything developers might want to do "out of the box." Because everything you need is part of the one "product," it all works seamlessly together, following consistent design principles.

The MTV Architecture

Django follows the Model-View-Template (MTV) architectural pattern, which is similar to the popular Model-View-Controller (MVC) pattern but with Django-specific terminology:

Model

Defines the data structure. Contains the essential fields and behaviors of the data you're storing.

View

Accepts web requests and returns web responses. Contains the business logic.

Template

Defines how the user sees the data. HTML with Django Template Language.

Request/Response Cycle

# Django Request/Response Flow:
    
    User Request → URLconf → View → Model (if needed) → Template → Response
    
    # 1. User makes request to /blog/post/1/
    # 2. URLconf routes to PostDetailView
    # 3. View queries Post model (ID=1)
    # 4. View renders post_detail.html template
    # 5. HTML response returned to user

Important Note

Don't confuse Django's "View" with MVC's "Controller" or Django's "Template" with MVC's "View." In Django, the View is closer to what other frameworks call a Controller, and the Template is closer to what they call a View.
